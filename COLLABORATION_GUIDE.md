# NEURALTWIN 협업 가이드

## Lovable 협업 기능

Lovable에서는 두 가지 방식으로 협업할 수 있습니다:

1. **특정 프로젝트에 초대** - 해당 프로젝트만 접근 가능
2. **전체 워크스페이스에 초대** - 워크스페이스의 모든 프로젝트 접근 가능

자세한 내용: [Lovable 협업 문서](https://docs.lovable.dev/features/collaboration)

## 1. 특정 프로젝트 초대

### 초대 방법
1. 프로젝트 에디터 우측 상단의 **Share** 버튼 클릭 (모바일에서는 "+" 아이콘)
2. 초대할 사람의 이메일 주소 입력
3. 접근 레벨 선택
4. 초대 이메일 발송

### 프로젝트 접근 레벨

#### Viewer (뷰어)
- **권한**: 읽기 전용
- **용도**: 프로젝트 검토, 진행 상황 확인
- **제한**: 코드 편집 불가, 배포 불가

#### Editor (에디터)
- **권한**: 프로젝트 편집 가능
- **용도**: 개발 작업, 코드 수정
- **제한**: 설정 변경 불가, 협업자 관리 불가

#### Admin (관리자)
- **권한**: 설정, 협업자, 배포 관리
- **용도**: 프로젝트 관리자, 팀 리더
- **제한**: 프로젝트 삭제 불가

#### Owner (소유자)
- **권한**: 모든 권한 (프로젝트 삭제 포함)
- **용도**: 프로젝트 생성자
- **참고**: 소유권은 이전할 수 없지만, Remix로 복사 가능

### 주요 특징
- 초대받은 사람은 **해당 프로젝트만** 볼 수 있습니다
- 모든 사용량은 워크스페이스의 크레딧에서 차감됩니다
- 협업자의 개인 크레딧은 사용되지 않습니다

## 2. 워크스페이스 초대

### 초대 방법
1. **Settings > People** 메뉴로 이동 (워크스페이스 설정)
2. 이메일 주소 입력 및 역할 선택
3. 워크스페이스의 모든 프로젝트 접근 권한 부여

### 워크스페이스 역할

#### Viewer (뷰어)
- 워크스페이스의 모든 프로젝트 조회 가능
- 편집 권한 없음

#### Editor (에디터)
- 워크스페이스의 모든 프로젝트 생성 및 편집 가능
- 워크스페이스 설정 변경 불가

#### Admin (관리자)
- 워크스페이스 설정, 결제, 멤버 관리 가능
- 워크스페이스 삭제 불가

#### Owner (소유자)
- 워크스페이스에 대한 완전한 제어 권한
- 워크스페이스 삭제 가능

#### Collaborator (협업자)
- 특수 역할 (자동 할당)
- 특정 프로젝트에만 초대된 사용자
- 초대받은 프로젝트만 접근 가능

## 협업 시나리오

### 시나리오 1: 외부 개발자와 협업

**상황**: 프리랜서 개발자가 NEURALTWIN의 Analytics 기능을 개발

**추천 방법**: 특정 프로젝트 초대 (Editor 권한)

**이유**:
- 다른 프로젝트는 보이지 않음
- 편집 권한만 부여
- 작업 완료 후 쉽게 권한 제거 가능

### 시나리오 2: 팀 전체 협업

**상황**: 회사의 전체 개발팀이 여러 프로젝트를 관리

**추천 방법**: 워크스페이스 초대 (Editor 또는 Admin 권한)

**이유**:
- 모든 프로젝트 접근 가능
- 새 프로젝트 생성 가능
- 팀원 관리 용이

### 시나리오 3: 클라이언트 리뷰

**상황**: 클라이언트가 개발 진행 상황을 확인

**추천 방법**: 특정 프로젝트 초대 (Viewer 권한)

**이유**:
- 읽기 전용 접근
- 실수로 코드 수정 방지
- 진행 상황만 확인 가능

### 시나리오 4: 디자이너 협업

**상황**: UI/UX 디자이너가 디자인 시스템을 검토하고 피드백 제공

**추천 방법**: 특정 프로젝트 초대 (Viewer 또는 Editor 권한)

**권한 선택**:
- **Viewer**: 디자인만 확인
- **Editor**: CSS/디자인 직접 수정

## 권한 관리 Best Practices

### 1. 최소 권한 원칙
필요한 최소한의 권한만 부여하세요.

```
클라이언트 리뷰 → Viewer
개발 작업 → Editor
프로젝트 관리 → Admin
```

### 2. 정기적인 권한 검토
- 월 1회 협업자 목록 확인
- 더 이상 필요 없는 권한 제거
- 역할 변경이 필요한지 검토

### 3. 프로젝트별 vs 워크스페이스
- **단기 프로젝트**: 프로젝트 초대
- **장기 팀원**: 워크스페이스 초대
- **외부 인력**: 프로젝트 초대 (기간 제한)

### 4. 문서화
협업자 목록과 역할을 문서화하세요:

```markdown
## 현재 협업자 (2024-01-15 기준)

### 프로젝트 초대
- john@example.com (Editor) - Frontend 개발
- sarah@design.com (Viewer) - UI/UX 검토

### 워크스페이스 멤버
- team@company.com (Editor) - 전체 팀
- manager@company.com (Admin) - 프로젝트 관리자
```

## FAQ

### Q: 한 프로젝트에만 초대할 수 있나요?
**A**: 네! Share 버튼을 사용하면 해당 프로젝트만 공유됩니다.

### Q: 모든 프로젝트 접근 권한을 주려면?
**A**: Settings > People에서 워크스페이스에 초대하세요.

### Q: 워크스페이스에 몇 명까지 초대 가능한가요?
**A**: 모든 플랜에서 무제한입니다. Enterprise 플랜은 커스텀 제한 설정 가능합니다.

### Q: 여러 워크스페이스를 가질 수 있나요?
**A**: 네, 하지만 각각 별도의 구독이 필요합니다.

### Q: 프로젝트를 다른 워크스페이스로 이전할 수 있나요?
**A**: 네, 프로젝트 설정에 Transfer 옵션이 있습니다.

### Q: 프로젝트 소유자를 변경할 수 있나요?
**A**: 소유권 이전은 불가능하지만, Remix로 프로젝트를 복사하여 새 소유자를 설정할 수 있습니다.

### Q: 협업자가 크레딧을 사용하나요?
**A**: 아니요. 모든 사용량은 워크스페이스 소유자의 크레딧에서 차감됩니다.

## NEURALTWIN 프로젝트 협업 예시

### 개발 팀 구성

```
워크스페이스: NEURALTWIN
├── Owner: ceo@neuraltwin.com
├── Admin: manager@neuraltwin.com
├── Editor: dev1@neuraltwin.com
├── Editor: dev2@neuraltwin.com
└── Viewer: stakeholder@neuraltwin.com

프로젝트 초대 (Analytics 기능)
├── Editor: contractor@freelance.com (외부 개발자)
└── Viewer: client@company.com (클라이언트)
```

### 역할별 작업

#### Owner / Admin
- Supabase 연동 설정
- 환경 변수 관리
- 배포 설정
- 팀원 권한 관리

#### Editor
- 코드 작성 및 수정
- 새 기능 개발
- 버그 수정
- PR 리뷰

#### Viewer
- 진행 상황 확인
- 디자인 검토
- 피드백 제공
- 테스트 (읽기 전용)

## 보안 권장사항

### 1. 민감한 정보 관리
- API 키는 Supabase Secrets에 저장
- 환경 변수는 Lovable 환경 설정에서 관리
- 코드에 하드코딩하지 않기

### 2. 접근 제어
- Supabase RLS 정책 활성화
- `user_roles` 테이블로 권한 관리
- Admin 역할 최소화

### 3. 정기 감사
- 분기별 협업자 권한 검토
- 사용하지 않는 계정 제거
- 로그 확인

## 문제 해결

### 협업자가 프로젝트를 볼 수 없음
1. 초대 이메일 확인 요청
2. 스팸 폴더 확인
3. 정확한 이메일 주소로 초대했는지 확인
4. 워크스페이스가 아닌 프로젝트 초대인지 확인

### 권한이 작동하지 않음
1. 브라우저 새로고침
2. 로그아웃 후 다시 로그인
3. 역할이 올바르게 설정되었는지 확인

### 크레딧 사용량이 많음
1. 협업자 수 확인
2. 불필요한 협업자 제거
3. 워크스페이스 플랜 업그레이드 고려
